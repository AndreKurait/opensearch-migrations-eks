## What You Get

<CardGrid>
  <Card title="Keep Production Running" icon="approve-check-circle">
    Migration reads from a snapshot in S3 — not your live cluster. After one snapshot, your source is never touched again. Zero performance impact.
  </Card>
  <Card title="Skip Intermediate Upgrades" icon="random">
    Go directly from Elasticsearch 1.x to OpenSearch 3.x. No stepping through 5.x → 6.x → 7.x → OS 1.x. One migration, any version gap.
  </Card>
  <Card title="Scale Without Limits" icon="up-caret">
    Each worker processes one shard independently from S3. Add workers to go faster — 590K docs/min per worker. Failed shards retry automatically.
  </Card>
  <Card title="Declarative Workflows" icon="puzzle">
    Define your migration as YAML. Argo Workflows handles orchestration with parallel execution, retry logic, and approval gates before critical steps.
  </Card>
</CardGrid>

## Before You Start

:::note[Prerequisites]
- **AWS account** with permissions to create EKS clusters, S3 buckets, and IAM roles
- **kubectl** and **Helm 3** installed locally
- **Source cluster** accessible from your VPC (Elasticsearch 1.x–8.x or OpenSearch 1.x–2.x)
- A **snapshot** in S3 — or the ability to create one
:::

## How It Works

<Steps>
1. **Deploy** (~10 min) — Run the bootstrap script to create an EKS cluster with Migration Assistant

2. **Configure** (~5 min) — Define source, target, and migration options in YAML from the Migration Console

3. **Migrate** (varies by data size) — Submit the workflow. It snapshots, migrates metadata, backfills documents, and pauses for your approval at each stage

4. **Validate & Switch** (~15 min) — Verify document counts and query results, then redirect traffic to the target
</Steps>

## What's Migrated

| ✓ Supported | ✗ Manual Setup Required |
|-------------|------------------------|
| Documents | ISM / ILM policies |
| Index settings & mappings | Security configuration |
| Index templates (legacy + composable) | Kibana / Dashboards objects |
| Component templates | Ingest pipelines |
| Aliases | Data streams |

## Version Support

| Source | → OpenSearch 1.x | → OpenSearch 2.x | → OpenSearch 3.x |
|--------|:-:|:-:|:-:|
| **Elasticsearch 1.x–7.x** | ✓ | ✓ | ✓ |
| **Elasticsearch 8.x** | — | ✓ | ✓ |
| **OpenSearch 1.x** | ✓ | ✓ | ✓ |
| **OpenSearch 2.x** | — | ✓ | ✓ |

Runs on: **Amazon EKS** · GKE · AKS · OpenShift · any Kubernetes 1.24+

<Aside type="tip">
Already deployed? Reconnect: `kubectl exec -it migration-console-0 -n ma -- bash` then run `workflow configure sample` to see the config schema for your version.
</Aside>
